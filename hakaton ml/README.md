# Хакатон: Предсказание результатов обучения студентов

## Описание задачи

Предсказание итоговых результатов обучения студента (выпустился или отчислен) по результатам его обучения за первые два академических курса.

## Структура проекта

```
.
├── data/
│   ├── raw/              # Исходные данные (CSV)
│   └── processed/        # Обработанные данные (Parquet)
├── src/
│   ├── load_data.py      # Загрузка и анализ данных
│   ├── preprocess.py     # Предобработка данных
│   ├── feature_engineering.py  # Создание признаков
│   ├── split_train_test.py    # Разделение на train/test
│   ├── baseline.py       # Baseline модель (логистическая регрессия)
│   ├── model_catboost.py # CatBoost модель
│   ├── model_lightgbm.py # LightGBM модель
│   ├── validation.py     # Валидация моделей
│   ├── inference.py      # Предсказания на тесте
│   └── utils.py          # Вспомогательные функции
├── models/               # Сохраненные модели
├── output/               # Результаты и логи
└── requirements.txt      # Зависимости
```

## Установка

```bash
pip install -r requirements.txt
```

## Использование

### 1. Загрузка и анализ данных

```bash
python src/load_data.py
```

### 2. Предобработка данных

```bash
python src/preprocess.py
```

### 3. Создание признаков

```bash
python src/feature_engineering.py
```

### 4. Разделение на train/test

```bash
python src/split_train_test.py
```

### 5. Обучение моделей

**Baseline:**
```bash
python src/baseline.py
```

**CatBoost:**
```bash
python src/model_catboost.py
```

**LightGBM:**
```bash
python src/model_lightgbm.py
```

### 6. Предсказания на тесте

```bash
python src/inference.py
```

Результат будет сохранен в `output/submission.csv`

## Логика разделения данных

- **Train**: студенты с данными за семестры 1-4 (первые 2 курса) + есть целевая переменная
- **Test**: студенты из `sample_submission.csv` (для них нужно предсказать результат)

## Признаки

Модель использует следующие группы признаков:

1. **Статистики по оценкам**: среднее, медиана, мин, макс, std
2. **Динамика по семестрам**: тренд оценок, улучшение/ухудшение
3. **Статистики по типам оценок**: экзамены, зачеты, практики, курсовые
4. **Статистики по дисциплинам**: количество уникальных дисциплин
5. **Процент успешных оценок**: success_rate, excellent_rate, fail_rate
6. **Информация о студенте**: факультет, направление, год поступления

## Модели

- **Baseline**: Логистическая регрессия с балансировкой классов
- **CatBoost**: Gradient boosting с категориальными признаками
- **LightGBM**: Gradient boosting с оптимизацией памяти

## Проверка на даталики

Все признаки проверяются на возможные утечки информации из будущего. Используются только данные за первые 2 курса (семестры 1-4).

## Логи

Все логи сохраняются в `output/logs.txt`
